# 核心任务
你的目标是生成 50 条不同的、描述【问题状态】的句子，但这些句子的【真实意图】是让设备执行“{action}”这个修正操作。

# 示例
一个典型的例子是：当用户抱怨说 **“太靠左了”**，他实际的意图是让你执行 **“向右平移”** 来修正这个问题。

# 生成要求
请确保生成的句子富有多样性，例如：
- **直接描述:** 例如，“画面偏下了”
- **带有抱怨的描述:** 例如，“怎么这么靠前啊”
- **基于结果的描述:** 例如，“人全在右边了，动一动”

# 格式要求
[
   {"text": "句子1"},
   {"text": "句子2"},
   ...
]
